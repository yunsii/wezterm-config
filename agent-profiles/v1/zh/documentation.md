# Documentation（中文对照）

Source of truth:
[../en/documentation.md](../en/documentation.md)

本文档是英文版的中文对照参考。
若中英文存在差异，以英文版为准。

当需要创建、拆分或修订面向 agent 的文档时，阅读本文件。

## 默认值

文档应降低决策成本，而不是变成第二套代码库。
保持分层、精简且易于导航。

## 分层

建议使用以下结构：

- 入口文档承载硬规则和路由
- 主题文档承载详细领域规则
- 局部文档承载环境或项目专属约束
- 参考文档仅在必要时承载深背景

不要把所有内容都放进入口文档。

## 渐进式披露

按当前任务只加载最小必要上下文。

起步时只读：
- 主入口
- 一份匹配的主题文件

只有在以下情况下再继续加载：
- 当前文档明确指向它
- 任务跨越多个边界
- 如果不读就会带来明显风险

## 写什么

好的文档应当：

- 具体
- 稳定
- 可执行
- 靠近决策点
- 易于扫读

每份主题文件理想上都应回答：

- 何时阅读
- 适用哪些规则
- 应优先什么
- 应避免什么
- 如何验证

## 避免

避免：

- 冗长的历史叙述
- 空泛口号
- 高频变化的工具琐事
- 多处重复定义同一规则
- 本应做成自动化的说明

## 维护

当某个文件变得过于宽泛时，应按决策领域拆分，而不是按受众心情拆分。
每条规则只保留一个真实来源。
其他文件只负责路由，不重复抄写。
