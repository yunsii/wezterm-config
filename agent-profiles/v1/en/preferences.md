# Preferences

This file contains personal preferences, not hard requirements.

Load this file only when it helps break ties between otherwise valid approaches.

## Default

Preferences must not override:

- project instructions
- correctness
- safety
- maintainability
- strong local conventions

## Use For

- preferred response brevity
- preferred amount of progress commentary
- naming taste when no project convention exists
- preferred verification order when several valid options exist
- favored tools when multiple tools are interchangeable

## Do Not Use For
- rules that should be hooks or scripts
- project-specific constraints
- unstable environment details
- anything that would create correctness risk if ignored
